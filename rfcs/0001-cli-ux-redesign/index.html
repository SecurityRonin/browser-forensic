    RFC 0001 — br4n6 CLI UX Redesign¶

    Status: Accepted (2026-07-13) — decisions settled; implementation begins at P0. Persona: an overworked, tired incident-response examiner at 2am. Mantra: don't make them think (Krug), by doing less magic and showing more provenance.

    Resolved on review: (1) default tier = --standard; (2) recover is an orchestrator over artifact primitives discoverable from its --help, no recover deleted/carve/memory submodes (the examiner asks "what's deleted?", not "carve vs memory"); (3) clean break — remove ALL backward compatibility, no aliases/shims (see D11); (4) ship the guarded bare-path convenience (D3, P7); (5) P3 includes full checkpoint/resumability for huge images. The bw short binary-name is retained (a convenience alias, not backward-compat).

    Summary¶

    br4n6 today exposes 46 flat subcommands. That is a wall: the examiner must already know the forensic taxonomy and this tool's vocabulary to pick one. This RFC proposes a two-layer surface — a small set of task-oriented verbs the examiner already thinks in (their questions), over the existing 46 primitives (kept as capabilities, reachable under an artifact <name> namespace). It is a clean break at a major version: the old flat command names are removed outright — no aliases, no compatibility shims (see D11).

    The design was generated by one model (Fable 5) and independently pressure-tested by two critics (Codex and Gemini 3.1 Pro). Their two convergent objections — a heavy default that looks crashed or hides evidence at 2am, and overclaiming labels that don't hold in court — shape the decisions below. Where the critics rejected an idea, this RFC rejects it too.

    Problem — why a tired examiner has to think¶

    # Problem Cost at 2am
    1 46 flat commands, no hierarchy --help is a wall; must know the tool's vocabulary to choose
    2 No "just tell me what happened" bare br4n6 <path> errors today — the most likely first invocation fails
    3 Synonym verb-sets (chains/correlate/graph; search/extract-iocs/match-domains; 4× cache*) wrong pick → fear of missed evidence
    4 PATH ambiguity (file? profile? image? home? dump?) examiner must classify their own evidence before the tool will
    5 Decryption flag soup (--decrypt-win --local-state --dpapi-masterkey) 3-flag, platform-specific incantation; silent encrypted blobs on failure
    6 6 Chromium micro-artifacts at top level must know Chromium internals to know they're usually skippable
    7 Output defaults to per-artifact row dumps answer buried in a wall
    8 Court-honesty is per-finding copy, not structural a renderer can still emit an absolute ("user deleted history")

    Design principles¶

    1. Verbs are the examiner's questions, not the format's nouns. What happened → investigate. Did they touch X → find. When → timeline. What's deleted → recover. What did it look like → reconstruct. What do I hand the lawyer → report.
    2. Do less magic, show more provenance. Every auto-decision (path type, key source, severity) is shown with its basis and confidence, never silently applied.
    3. Bounded by default, exhaustive on request. The default must never look crashed (Gemini's "black hole") and never silently skip the evidence they fear missing (Codex's "false reassurance").
    4. Court-safe by construction. Priority ≠ confidence ≠ interpretation, enforced by the data model — not by careful wording a future renderer could violate.
    5. Clean break — the new surface is the only surface. The 46 primitives live on as capabilities under artifact <name>, but the old flat command names are removed at a major version. No aliases, no shims: one honest way to do each thing, and migration is a one-time doc, not permanent code.

    The redesigned surface¶

    br4n6 investigate <PATH>     # bounded triage → ranked, provenance-tagged summary   (canonical)
br4n6 find <TERM> <PATH>     # "did they visit/download/search X?" — provenance-first
br4n6 timeline <PATH>        # unified chronology across artifacts
br4n6 recover <PATH>         # deleted/carved/tamper — orchestrator with explicit submodes
br4n6 reconstruct <URL> <PATH>  # cached representations consistent with access (html/warc/gallery)
br4n6 report <PATH>          # reproducible court/exam bundle (html+xlsx+jsonl+manifest+hashes)

# power / discovery layer
br4n6 artifact <NAME> <PATH> # the 46 primitives, one namespace, current semantics + --format
br4n6 artifact --list        # table: name · browser family · what it proves
br4n6 ls <PATH>              # instant inventory: browsers, profiles, users, artifact counts
br4n6 tui <PATH>             # interactive viewer (unchanged)
br4n6 schema [NAME]          # BrowserEvent JSON Schema (unchanged)

br4n6 <PATH>                 # CONVENIENCE ONLY → investigate, guarded (see D3)
br4n6                        # no args on a TTY → tldr-style cheat sheet, then exit

    Decisions (each carries the critic finding it resolves)¶

    D1 — Task-verbs are the default help surface; artifact preserves all power¶

    Default --help shows the 6 verbs + ls/tui. The 46 primitives are reachable, discoverable from br4n6 artifact --list and from each verb's --help "Next:" pointers — not hidden so well experts can't find artifact-specific workflows (Codex). Build as-is (unanimous).

    D2 — Bounded tiering is the default; skipped work is impossible to miss¶

    Both critics rejected a heavy investigate as the zero-thought default (Gemini: looks crashed; Codex: hides deleted evidence). Instead:

    --quick     live artifacts + cheap integrity
--standard  quick + bounded deleted-SQLite/WAL recovery        ← DEFAULT
--deep      standard + carving + cache reconstruction + memory + disk-wide scans

    Bare investigate runs --standard. The summary footer always states what it did not do: Deep recovery not run: carving, cache reconstruction, memory skipped. Deleted evidence may be missed → br4n6 investigate --deep <PATH>. Plus progress heartbeats (indicatif: phase, rows, ETA) and Ctrl-C flushes the partial ranked summary and exits clean (a silent terminal is a dead terminal — Gemini).

    D3 — Bare br4n6 <PATH> is convenience-only, ambiguity-guarded¶

    investigate <PATH> is the canonical, documented form. Bare path is accepted only when the single token exists on disk and is not an exact command name; a token that is both a command and a path fails with a specific diagnostic: 

    Ambiguous: "find" is both a br4n6 command and a path.
Use: br4n6 investigate ./find   |   or: br4n6 find <term> <path>
    Test matrix: paths named after every verb, paths beginning with -, -- passthrough. "PATH always last" is rejected as a rigid rule; support -- and explicit --term/--regex/--terms-file for the awkward cases (Codex).

    D4 — find preserves provenance; it does NOT homogenize evidence classes¶

    Collapsing search/extract-iocs/match-domains/recovered-domains into one verb is fine as a front door, lossy as a data model. A live history hit, a carved string, and a recovered deleted domain have different confidence and courtroom value. Default find output carries columns:

    TERM       SOURCE    STATE       CONF  TIME BASIS   USER-ACTION       MATCH
evil.com   history   live        high  explicit     visited           https://evil.com/a
evil.com   cache     live        med   surrounding  observed-string   https://evil.com/a.js
evil.com   carved    deleted     low   none         unknown           <fragment>
    source · state(live/deleted/carved/reconstructed/inferred) · confidence(+rule id) · timestamp_basis · user_action_claim are structural fields, present in text and JSONL.

    D5 — Split Priority / Confidence / Interpretation (kill the bare "HIGH")¶

    A single HIGH reads as high confidence of wrongdoing in a legal/exec context. Each finding renders three separate axes, and the split is enforced by the finding data model so no renderer can emit an absolute: 

    Priority:       High            (look here first — a triage attention cue)
Confidence:     Medium          (+ rule id)
Interpretation: consistent with manual deletion, DB maintenance, or profile sync
Rule:           integrity.history.rowid_gap.v1
Evidence:       Chrome History urls rowid gap 128 → 944
Next:           br4n6 artifact integrity --rule history.rowid_gap <PATH>
    A one-line report footer states once that Priority is an attention cue, not a finding of malice.

    D6 — reconstruct, not "show what the user SAW"¶

    Cache shows what was stored, not what was rendered, foregrounded, or seen by a human. The verb is reconstruct; output is phrased "cached representations consistent with access to <url>", with the existing found-vs-missing sub-resource provenance manifest. (Rename also sets the "heavy operation" expectation — Gemini.)

    D7 — --keys is evidence-root-constrained; secrets are output-oriented; keys are audited¶

    One-flag --keys <PATH> (both critics' favorite idea) — but auto-locating keys is a contamination/attribution hazard (Codex). Rules: - Never search outside the evidence root unless --keys <external> is given explicitly. - Print exactly which key sources were used; hash/identify every key file into the manifest. - Distinguish "key found" from "key successfully decrypted this item." - Without keys: encrypted material is counted and reported, never silently dropped (1,022 cookies encrypted — add --keys <path>). - Passwords render [decrypted — write to file with --reveal-secrets <file>]. --reveal-secrets is file-output-oriented, never terminal by default (scrollback/tmux/screen-share leak PII). Password input via --password-stdin / prompt, never user:pw on argv (leaks to shell history + ps).

    D8 — Auto-detect with layered confidence, logged to the manifest¶

    Magic bytes alone say "SQLite," not "Chrome History vs Cookies vs Firefox places." Detection is layered (magic → schema probe → directory markers → container/partition probe) and every result reports basis + confidence: 

    Detected: Chromium History (SQLite)   Confidence: high
Basis:    SQLite header + urls/visits schema + parent path .../Default/History
    Memory-dump and raw-disk detection are explicitly low-confidence unless a known header/container is present. The detection basis is written into the manifest for court defensibility. A --type <kind> override always exists (Gemini: the tool will guess wrong on carved/stomped data).

    D9 — Multi-user / multi-profile selectors; every finding stamped with origin¶

    "2 users, 3 browsers" is not enough — the examiner needs to know which user/profile each finding came from. Stable selectors: --user <SID|name>, --profile "Chrome/Default", --browser chrome. Every event/finding carries its user/profile/browser provenance in text and JSONL.

    D10 — Output UX¶

    • TTY → ranked summary render; pipe → JSONL (isatty(stdout)), --json/--plain force. On a pipe, stderr announces loudly: [notice] piped output → JSONL; use --format summary to override (Gemini: a silent schema switch confuses tee/grep users).
    • No box-drawing characters — markdown-clean tables that survive paste into Jira/Word/ServiceNow (Gemini; matches the fleet no-box-drawing rule). Severity word always printed (survives tee); color is a TTY-only cue, NO_COLOR honored.
    • Full values never ellipsized (URLs/hashes/GUIDs) — truncated evidence is destroyed evidence.
    • Negative-result discipline: find X with no hits says where it looked and what it skipped — no hits in live history/downloads/bookmarks; skipped: encrypted cookies, memory, carving. An empty result must prove it looked.
    • Actionable errors: a locked/dirty-WAL DB suggests the fix (try: br4n6 recover <PATH> --ignore-wal), never a bare SQLITE_CORRUPT.

    D11 — Auditability is first-class; the redesign is a clean break (no backward compat)¶

    Every run records into a manifest: exact command line, tool + rule + parser versions + build hash, evidence path identity + hashes, detection basis + confidence, skipped sources + failed parsers, timezone source + conversion rule, redaction policy, output schema version.

    Migration is a clean break at a major version — no aliases, no compatibility shims. The old flat command names (br4n6 history, chains, predictor, recovered-domains, …) are removed; their capability is reachable only via the new verbs or artifact <name>. Rationale (owner decision on review): this is a solo-owned tool at the right moment to break — before an external user base ossifies around the old strings — and permanent alias plumbing (46 shims + a byte-identical golden test each, forever) is real cost for a compatibility promise no external lab has yet relied on. The accepted tradeoff: any existing script or SOP citing an old command breaks at the version bump. Mitigation: a one-time migration guide (docs/migration-v2.md, delivered in P8) mapping every old command → its new form — a document, not runtime code. The bw short binary-name is retained (a convenience alias for br4n6, not backward-compat).

    Rejected (both critics)¶

    • Bare-path as the core contract (D3 demotes it to guarded convenience).
    • "show what the user SAW" wording (D6).
    • Auto-locating keys from broad system locations (D7 constrains to evidence root).
    • A one-screen ranked summary that can imply completeness (D2 footer + D5 axes).
    • A guided TUI menu on bare br4n6 (→ tldr cheat sheet + exit).
    • Hiding primitives so well experts can't discover artifact workflows (D1).
    • "PATH always last" as a rigid rule (D3).
    • Backward-compat aliases / shims (superseded on review) — the earlier draft kept the 46 old names as hidden aliases forever; the accepted decision is a clean break with a one-time migration guide instead (D11).

    Phased implementation plan¶

    Each phase is independently shippable, strict-TDD, non-breaking, and lands green. No phase removes a command.

    Phase Deliverable Acceptance Risk
    P0 — Foundations (no user-visible change) The finding/provenance data model: Finding { priority, confidence(+rule_id), interpretation, evidence, source, state, timestamp_basis, user_action_claim, user/profile/browser }; the manifest gains detection-basis + skipped-sources + versions fields. Rendered nowhere new yet. New types + JSON Schema; existing outputs unchanged; round-trip tests Low — additive
    P1 — artifact namespace + clean break br4n6 artifact <name> <PATH> wrapping all 46 primitives; the old flat command names are removed (no aliases). artifact --list; default --help shows only verbs+ls+tui. 46 primitives reachable via artifact; old flat names gone; help shows only the new surface Low — routing + deletions
    P2 — Output engine isatty summary/JSONL split + loud stderr notice; markdown-clean tables (no box-drawing); full-value rendering; NO_COLOR; negative-result discipline; actionable errors. Applied to existing verbs first. Snapshot tests TTY vs pipe; paste-survival check; find-no-hits shows skipped-sources Med — touches every renderer
    P3 — investigate tiering + summary --quick/--standard/--deep, default standard; ranked summary from the P0 finding model with the split axes + skipped-work footer; progress heartbeats + Ctrl-C partial flush. Tiers do bounded work (timed on a large fixture); footer always lists skipped; SIGINT flushes High — the core, and the streaming/interrupt engine
    P4 — find with provenance Auto-classify TERM (domain/url/ip/hash/regex/@file), search across sources, provenance columns; negative-result discipline. Absorbs search/extract-iocs/match-domains/recovered-domains as one front door over distinct data. Provenance columns in text+JSONL; recovered ≠ live never merged; -- + --term/--regex Med
    P5 — timeline / recover / reconstruct timeline (chronology + --around); recover as a single orchestrator that runs all recovery (deleted records, carving, cache carve, memory, tamper) and ranks results — no submodes; the underlying primitives stay discoverable from recover --help (as artifact pointers) for specialists. reconstruct (renamed, "consistent with access" phrasing). recover <PATH> runs everything, one call, no mode choice; primitives discoverable but not required; reconstruct wording; timeline --around Med
    P6 — --keys decryption UX One-flag --keys, evidence-root-constrained auto-locate, key sources hashed→manifest, "found vs decrypted" distinction, --reveal-secrets <file> (file-only), --password-stdin. Constrained search (test it won't read outside root); manifest logs key hashes; no secret to TTY by default Med — security-critical
    P7 — Auto-detect + selectors + bare-path Layered detection with confidence + basis→manifest + --type override; --user/--profile/--browser selectors; guarded bare-path convenience with the ambiguity diagnostic + full test matrix. Detection basis shown+logged; ambiguity tests for verb-named paths; selectors stamp findings Med
    P8 — report bundle + docs + migration guide Reproducible report (html+xlsx+jsonl+manifest+hashes, one shot); README rewritten around the 6 verbs; docs/migration-v2.md mapping every removed old command → its new form; shell completions. One command → full bundle + SHA sidecar; migration guide covers all 46 removed names; docs; completions Low

    Sequencing rationale: P0 (data model) and P2 (honest rendering) de-risk the hard P3 streaming/tiering core. P1 is the clean-break cut-over — after it lands, the tool speaks only the 6-verb + artifact surface. Because it's a clean break (not an additive alias layer), P1 is a single decisive version bump rather than a compatibility-preserving shim, and the migration guide (P8) is the safety net for anyone with old scripts.

    Full checkpoint/resumability (P3): huge-image runs write a resumable checkpoint (parsed-so-far state + a content-addressed progress marker) so a killed or crashed investigate/report on a multi-TB image resumes from the last completed unit rather than restarting — over and above the SIGINT-flush-partial-summary behavior.

    Resolved decisions (owner review, 2026-07-13)¶

    1. Default tier = --standard — bounded deleted-SQLite/WAL recovery in the zero-thought default; whole-image carving stays opt-in (--deep). The --standard footer always names what it skipped, so speed and the "did I miss deleted evidence" fear are both addressed.
    2. recover has no submodes — one orchestrator runs all recovery and ranks results; the examiner never chooses carve-vs-memory-vs-WAL. Underlying primitives remain discoverable from recover --help for specialists.
    3. Clean break, no backward compat — old flat command names removed; no aliases/shims; one-time docs/migration-v2.md (P8) instead. bw short-name retained. (D11.)
    4. Bare-path shipped, guarded — br4n6 <PATH> runs investigate, accepted only when the token is an on-disk path and not a command name, with the ambiguity diagnostic + full test matrix (D3, P7).
    5. P3 = full checkpoint/resumability now — resumable checkpoints for multi-TB images, not just SIGINT-flush.

    Non-goals¶

    • Rewriting the parsers or the forensic logic (this is a UX/CLI-surface RFC).
    • A GUI (the TUI stays as-is).
    • Removing any capability.

    Process note: design by Fable 5; adversarial critique by Codex; independent second opinion by Gemini 3.1 Pro. The two critics independently converged on the heavy-default and overclaiming-label objections, which drove D2 and D5/D6.
